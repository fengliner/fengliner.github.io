Mongodb解决不能连接到服务器的错误

mongodb出现以下错误： couldn’t connect to server 127.0.0.1

解决方法为：

先看服务器启动没有，如果没有启动服务先启动服务，如果服务已经启动了，可能是由于上次不正常关闭mongo，导致存放数据的文件被占用了，被锁住了。只需将E:\software\mongodb\data(根据自己的安装路径而定)中的mongod.lock文件删除掉。重新启动服务即可。
